## What are Fragments?

Fragments are executable code packages generated by AI agents. They contain the complete code structure, files, and execution environment needed to run a specific application or feature.

## Fragment Structure

Each fragment includes:

- **Files**: Complete code structure with all necessary files
- **Sandbox URL**: Live preview of the running application
- **Title**: Descriptive name for the fragment
- **Message Link**: Associated with the message that generated it

## Database Schema

Fragments are stored with the following structure:

```sql
model Fragment {
  id         String   @id @default(uuid())
  messageId  String   @unique
  message    Message  @relation(fields: [messageId], references: [id])
  
  sandboxUrl String
  title      String
  files      Json
  
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt
}
```

## Fragment Generation

### AI Agent Process

When a user message is processed, the AI agent:

1. **Analyzes the request** and determines what code to generate
2. **Creates the file structure** with all necessary files
3. **Executes the code** in an E2B sandbox
4. **Generates a live preview** URL
5. **Stores the fragment** in the database

<CodeGroup>

```typescript Fragment Creation
const result = await network.run(userMessage);

const fragment = await prisma.message.create({
  data: {
    projectId: event.data.projectId,
    content: result.state.data.summary,
    role: "ASSISTANT",
    type: "RESULT",
    fragment: {
      create: {
        sandboxUrl: sandboxUrl,
        title: "Todo Application",
        files: result.state.data.files
      }
    }
  }
});
```

</CodeGroup>

## Viewing Fragments

### In the UI

Fragments are displayed in the project interface with:

- **Preview Tab**: Live preview of the running application
- **Code Tab**: File explorer showing the complete code structure
- **Sandbox URL**: Direct link to the running application

<CodeGroup>

```typescript Fragment Display
export function FragmentView({ fragment }: { fragment: Fragment }) {
  return (
    <Tabs defaultValue="preview">
      <TabsList>
        <TabsTrigger value="preview">Preview</TabsTrigger>
        <TabsTrigger value="code">Code</TabsTrigger>
      </TabsList>
      
      <TabsContent value="preview">
        <iframe 
          src={fragment.sandboxUrl} 
          className="w-full h-full border-0"
        />
      </TabsContent>
      
      <TabsContent value="code">
        <FileExplorer files={fragment.files} />
      </TabsContent>
    </Tabs>
  );
}
```

</CodeGroup>

## Fragment Limitations

### Size Limits
- Maximum total file size: 10MB per fragment
- Maximum individual file size: 1MB
- Maximum number of files: 1000

### Execution Limits
- Sandbox timeout: 2 hours
- Memory limit: 2GB
- CPU limit: 2 cores
- Network restrictions apply

## Best Practices

### File Organization

│   └── features/

- Use TypeScript for type safety
- Include proper error handling
- Add comments for complex logic
- Follow consistent naming conventions

### Performance

- Minimize bundle size
- Use code splitting where appropriate
- Optimize images and assets
- Implement lazy loading

## Fragment Persistence

### Storage Duration
- Fragments are stored for 30 days
- Sandbox URLs expire after 2 hours of inactivity
- Files can be downloaded before expiration

### Backup and Export

<CodeGroup>

```typescript Export Fragment
export async function downloadFragment(fragmentId: string) {
  const fragment = await trpc.fragments.get.query({ id: fragmentId });
  
  // Create ZIP file with all files
  const zip = new JSZip();
  
  Object.entries(fragment.files).forEach(([path, content]) => {
    zip.file(path, content);
  });
  
  const blob = await zip.generateAsync({ type: "blob" });
  saveAs(blob, `${fragment.title}.zip`);
}
```

</CodeGroup>

## Common Issues

### Sandbox Timeout

If a sandbox times out:
1. Check if the application is still running
2. Refresh the preview to restart
3. Re-run the code generation if needed

### File Missing

If files are missing from the fragment:
1. Check the original message for completeness
2. Ask the AI to regenerate missing files
3. Verify the file paths are correct

### Build Errors

If the code doesn't build:
1. Check for syntax errors in the code
2. Verify dependencies are correctly specified
3. Review the build logs in the terminal
