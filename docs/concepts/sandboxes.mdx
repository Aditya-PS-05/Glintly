## What are Sandboxes?

Sandboxes are isolated, secure environments where generated code is executed. NovaCraft uses [E2B](https://e2b.dev) sandboxes to provide safe, containerized environments for running user-generated code.

## Sandbox Features

### Security & Isolation
- **Containerized environments** prevent code from accessing host system
- **Network restrictions** limit external access to approved domains
- **Resource limits** prevent resource exhaustion attacks
- **Automatic cleanup** removes environments after timeout

### Development Environment
- **Pre-configured templates** with common tools and runtimes
- **Package manager support** for installing dependencies
- **Terminal access** for running commands
- **File system access** for reading/writing files

## Sandbox Lifecycle

### 1. Creation
When a user sends a message, a new sandbox is created:

<CodeGroup>

```typescript Sandbox Creation
const sandboxId = await step.run("get-sandbox-id", async () => {
  const sandbox = await Sandbox.create("novacraft-nextjs-test");
  return sandbox.sandboxId;
});
```

</CodeGroup>

### 2. Code Execution
The AI agent executes code in the sandbox:

<CodeGroup>

```typescript Code Execution
// Write files to sandbox
await sandbox.files.write("src/App.tsx", appContent);
await sandbox.files.write("package.json", packageJson);

// Run terminal commands
await sandbox.commands.run("npm install");
await sandbox.commands.run("npm run build");
await sandbox.commands.run("npm start");
```

</CodeGroup>

### 3. Preview Generation
A live preview URL is generated:

<CodeGroup>

```typescript Preview URL
const sandboxUrl = await step.run("get-sandbox-url", async () => {
  const sandbox = await getSandbox(sandboxId);
  const host = sandbox.getHost(3000);
  return `https://${host}`;
});
```

</CodeGroup>

### 4. Timeout & Cleanup
Sandboxes automatically cleanup after inactivity:
- **2 hours** of inactivity triggers cleanup
- **Resources are released** back to the pool
- **Preview URLs** become inactive

## Sandbox Tools

### Terminal Access

Execute commands in the sandbox:

<CodeGroup>

```typescript Terminal Tool
createTool({
  name: "terminal",
  description: "Execute terminal commands in the sandbox",
  parameters: z.object({
    command: z.string(),
  }),
  handler: async ({ command }, { step }) => {
    const sandbox = await getSandbox(sandboxId);
    const result = await sandbox.commands.run(command, {
      onStdout: (data) => console.log(data),
      onStderr: (data) => console.error(data),
    });
    return result.stdout;
  }
});
```

</CodeGroup>

### File Operations

Read and write files in the sandbox:

<CodeGroup>

```typescript File Tools
// Write files
createTool({
  name: "createOrUpdateFiles",
  parameters: z.object({
    files: z.array(z.object({
      path: z.string(),
      content: z.string(),
    }))
  }),
  handler: async ({ files }) => {
    const sandbox = await getSandbox(sandboxId);
    for (const file of files) {
      await sandbox.files.write(file.path, file.content);
    }
  }
});

// Read files
createTool({
  name: "readFiles",
  parameters: z.object({
    files: z.array(z.string()),
  }),
  handler: async ({ files }) => {
    const sandbox = await getSandbox(sandboxId);
    const contents = [];
    for (const file of files) {
      const content = await sandbox.files.read(file);
      contents.push({ path: file, content });
    }
    return contents;
  }
});
```

</CodeGroup>

## Resource Limits

### Compute Resources
- **CPU**: 2 vCPU cores
- **Memory**: 2GB RAM
- **Storage**: 10GB disk space
- **Network**: 100Mbps bandwidth

### Time Limits
- **Execution timeout**: 30 seconds per command
- **Sandbox lifetime**: 2 hours maximum
- **Idle timeout**: 15 minutes of inactivity

### Usage Quotas
- **Concurrent sandboxes**: 10 per user
- **Daily sandbox hours**: 24 hours
- **Monthly sandbox hours**: 720 hours

## Best Practices

### Performance Optimization

<Tip>
Keep sandbox operations efficient to avoid timeouts and resource limits.
</Tip>

- **Minimize dependencies** in package.json
- **Use caching** for repeated operations
- **Batch file operations** instead of individual writes
- **Clean up temporary files** regularly

### Error Handling

<CodeGroup>

```typescript Error Handling
const result = await sandbox.commands.run(command, {
  onStdout: (data) => buffers.stdout += data,
  onStderr: (data) => buffers.stderr += data,
  timeout: 30000 // 30 second timeout
});

if (result.exitCode !== 0) {
  throw new Error(`Command failed: ${buffers.stderr}`);
}
```

</CodeGroup>

### Security Considerations

- **Validate user input** before executing commands
- **Sanitize file paths** to prevent directory traversal
- **Limit network access** to required domains only
- **Monitor resource usage** to prevent abuse

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Sandbox Creation Failed">
    **Possible causes:**
    - E2B API key is invalid
    - Template doesn't exist
    - Quota limits exceeded
    
    **Solutions:**
    - Verify API key configuration
    - Check template name and availability
    - Monitor usage quotas
  </Accordion>
  
  <Accordion title="Command Timeout">
    **Possible causes:**
    - Command takes too long to execute
    - Infinite loops in code
    - Network operations hanging
    
    **Solutions:**
    - Optimize slow operations
    - Add proper error handling
    - Use shorter timeouts for testing
  </Accordion>
  
  <Accordion title="File Not Found">
    **Possible causes:**
    - Incorrect file path
    - File wasn't created successfully
    - Permission issues
    
    **Solutions:**
    - Verify file paths are correct
    - Check file creation logs
    - Ensure proper permissions
  </Accordion>
  
  <Accordion title="Preview Not Loading">
    **Possible causes:**
    - Application failed to start
    - Wrong port configuration
    - Build errors
    
    **Solutions:**
    - Check application logs
    - Verify port 3000 is used
    - Review build output
  </Accordion>
</AccordionGroup>
